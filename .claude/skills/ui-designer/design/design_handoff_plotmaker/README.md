# Handoff: PlotMaker — LLMシナリオ作成プロットエディタ

## 概要
PlotMaker は、LLM にシナリオを生成してもらうためのプロットエディタ。
ユーザーはキャラクターを登録し、大雑把なシナリオ本文・メタ情報を入力すると、
LLM が続きを生成する。さらに最大 3 人のキャラクターを選んで相関図 (React Flow)
を描ける。

全体のトーンは **YouTube 風のフラットでシンプル** (左サイドバー + 上部ヘッダー)。

## このハンドオフ内のファイルについて
同梱の HTML ファイルは **デザインリファレンス** です。プロダクション用のコードとして
そのまま流用するものではありません。対象コードベースの **既存の環境 (Next.js +
React + Tailwind / Vite + React / SwiftUI など)** に、既存のコンポーネントライブラリ
と設計パターンを使って **レイアウトと挙動を再現** してください。対象環境がまだない
場合は最も適切な技術スタックを選んでください。React Flow は相関図画面で必須です。

## 忠実度
**低忠実度 (ワイヤーフレーム)** です。手書き風フォント (Kalam) + 整った枠線 +
スケッチ風プレースホルダで「構造と遷移」を示しています。
**色やフォントはそのまま採用せず、対象コードベースの既存デザインシステム** を適用
してください。本ハンドオフではレイアウト・情報設計・ユーザーフローを最優先で
再現することを目的とします。

## 画面 / ビュー

### 1. ダッシュボード / ホーム
- **目的**: ログイン直後のランディング。最近の作業物と新規作成導線を提供。
- **バリエーション提示**: A (クイック作成 + 最近), B (サムネグリッド — YouTube風),
  C (タイムライン + AI サジェスト)。
- **推奨**: B (グリッド型) を主、C のサジェスト要素を右サイドバーに統合。
- **レイアウト**: 上部ヘッダー (44px) + 左サイドバー (180px 展開時 / 56px 畳み時) +
  メインエリア。メインは 16-24px のパディング。
- **要素**:
  - フィルタチップ行 (すべて / 進行中 / 完結 / 下書き / キャラ多め / AI生成)
  - シナリオカードグリッド (4 列、カバー画像 + タイトル + ジャンル + 更新日 + キャラ数)
  - フローティング作成 FAB (右下)

### 2. キャラクター一覧
- **目的**: 登録済みキャラクターを俯瞰し、作成・編集へ遷移。
- **バリエーション**: A (カードグリッド 4 列), B (リスト + 右プレビュー), C (テーブル)。
- **推奨**: A をデフォルト、B/C はビュー切替で提供。
- **要素**:
  - アバター (円形 42x42)、名前、年齢、性別、役割タグ、性格 1 行
  - 右上「+ 新規作成」ボタン
  - 役割フィルタチップ (主人公 / ヒロイン / ライバル / サブ / 大人)

### 3. キャラクター作成 / 編集
- **目的**: キャラクターのプロパティを入力。
- **入力項目 (確定)**: 名前 / 年齢 / 性別 / 性格 / 外見 / 背景・経歴 / 口調・話し方 /
  目標・動機 (+ 任意のアバター画像)
- **バリエーション**: A (縦スクロール全フォーム), B (2 ペイン: 編集 + ライブプレビュー),
  C (タブ分け + AI 対話補助)。
- **推奨**: B。右ペインでキャラの "見た目" が即時プレビューされる安心感。
- **「AI で埋める」ボタン**: 空欄を LLM で補完。

### 4. シナリオ作成
- **目的**: 大雑把なあらすじ + メタを入力し、LLM で生成。
- **入力項目 (確定)**: タイトル / 舞台・世界観 / あらすじ (本文) / 参考作品
- **バリエーション**: A (本文エディタ中心 + 右サイドにメタ), B (メタカード + 生成パネル),
  C (章立て + 本文 + 起承転結アウトライン)。
- **推奨**: A。ヘッダー右に **生成** ボタン (⚡ アイコン)。押下で LLM にプロンプト送信し、
  本文末尾に続きを挿入 or 別ペインに提案として表示。
- **右パネル**: 登場キャラクターの紐付け (チップ形式、キャラ一覧から選ぶ)。
- **メンション記法**: `@キャラ名`・`#場所` でオートコンプリート。

### 5. キャラクター相関図 (React Flow)
- **目的**: 選択した最大 3 人のキャラ間の関係性をフロー図で記述。
- **バリエーション**: A (左ペインで選択 + 中央 Flow + 右でエッジ編集),
  B (フルキャンバス自由配置), C (関係性プリセット色分け + 凡例)。
- **推奨**: A を基本、C の関係性プリセット (恋愛 / 敵対 / 盟友 / 家族 / 不信) を採用。
- **要素**:
  - カスタムノード: 円形アバター + 名前 + 役割
  - エッジ: カーブ付、中央にラベル、色 = 関係性、方向矢印 (片方向 / 双方向)
  - 左ペイン: 選択中 (最大 3 人)、レイアウト切替 (三角 / 一列 / 放射)
  - 右ペイン: 選択中のエッジの詳細編集 (ラベル / 方向 / 強度 / メモ)
  - Flow コントロール (ズーム / フィット / グリッド)、ミニマップ右下

## グローバル構造 (全画面共通)
- **上部ヘッダー** (高さ 44px): メニュー開閉アイコン、ロゴ ▶ PlotMaker、検索、通知、
  アバター。
- **左サイドバー** (展開 180px / 畳み 56px): ホーム / キャラクター / シナリオ / 相関図 /
  設定。展開時は下部に「最近のシナリオ」リストも表示。
- **ダーク / ライトテーマ** 切替をサポート (CSS 変数ベース推奨)。

## インタラクション & ビヘイビア
- サイドバーのトグル (ヘッダー左のハンバーガー)
- シナリオの LLM 生成: 「生成」ボタン → ローディング → ストリーミング差分表示
- 相関図はドラッグでノード移動、エッジクリックで右ペインに詳細
- キャラ / シナリオ作成は 500ms 程度のデバウンスで自動保存
- 空欄を LLM で自動補完するボタン (キャラ編集 / シナリオ)
- 一覧系はクリックで詳細遷移、ホバーで軽い背景色変化

## ステート管理の目安
- `characters[]`: { id, name, age, gender, personality, appearance, background,
  speechStyle, goal, avatarUrl, tags[] }
- `scenarios[]`: { id, title, setting, synopsis, references[], chapters[],
  linkedCharacterIds[] }
- `relations[]`: { id, scenarioId, nodes[{charId, x, y}], edges[{from, to,
  label, type, direction, strength, memo}] }
- `ui`: { sidebarCollapsed, theme, activeScreen }
- LLM 生成は async ストリーム (Server-Sent Events / fetch stream)

## デザイントークン (参考値 — 既存 DS があればそちらを優先)
- 色: 背景 `#faf8f3` / 線 `#1f1a16` / 弱線 `#cbc2b5` / アクセント赤茶 `#c96442` /
  ハイライト黄 `#fff2a8`
- 角丸: カード 4px / ボタン 20px (ピル) / アバター 円
- タイポ: 見出しはやや大きめ (20-26px, 600)、本文 12-14px
- スペーシング: 8 / 12 / 14 / 18 / 24 / 28px

## アセット
- アイコンは Lucide / Heroicons など既存の SVG アイコンセットを推奨 (本ワイヤーでは
  自作 SVG のため流用不要)
- React Flow: `reactflow` パッケージ
- フォントは対象コードベースの既存 DS に従う (Kalam はワイヤー用)
